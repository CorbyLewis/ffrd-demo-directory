# EVENT-VALIDATION
This folder stores the complete model and final results of replication of each historic event through the full model suite that has been prepared for cloud compute. These model simulations act as a QC step to ensure the fully linked system of models still reproduce historic event results. 

## Event Validation Runs should:
* Validate linked models reproduce observed calibration events sufficiently on a desktop environment
* Prepare the model for reproduction of the calibration events in a cloud environment
* Have 1 person responsible for these scenarios, including desktop simulation.
* No use of observed data (other than as reference data for comparison to modeled results)
* SST-ready naming conventions, links, etc
* upload entire directory (all models: HMS, ResSim, RAS) after validation on desktop
* no cloud-compute involved
